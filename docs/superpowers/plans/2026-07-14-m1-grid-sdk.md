# M1 Grid SDK Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the encoder’s production grid surface with standards-based Geohash, UTM/UPS MGRS plus stable topology addresses, and a repository-owned pure-Python ISEA4H implementation aligned with DGGRID v8.44, then expose the frozen contract through the SDK, encoder API, and Web facade.

**Architecture:** Freeze one breaking shared contract first: requests carry `requested_grid_level`, while every result carries its own `grid_level` through `GridAddress`; MGRS additionally carries `topology_code`. Implement the three engines behind that contract in isolated, non-overlapping worktrees, validate replacements before deleting S2/Tile Matrix/Plane Grid/H3 from `cube_encoder`, then integrate reviewed slice commits in one fixed `git cherry-pick --no-commit` order before the one M1 commit. ISEA4H runtime code is pure Python; DGGRID is used only to generate and optionally re-check committed fixtures.

**Tech Stack:** Python 3.11, Pydantic 2, FastAPI, Shapely 2, PyProj 3.5, `mgrs` 1.4.6, pytest 8, Hypothesis, rasterio for the independent real-AOI gate, Vue 3/Vite for the encoding facade, optional DGGRID v8.44 development binary.

## Global Constraints

- Normative source: `docs/superpowers/specs/2026-07-13-partition-dataset-quality-refactor-design.md`; when another document conflicts, this source wins.
- This is a breaking development refactor: do not retain compatibility request fields, response fields, enum members, ST prefixes, engine aliases, or legacy parsing.
- Production `grid_type` is exactly `geohash | mgrs | isea4h`; derived partition methods are `logical | logical | entity` respectively.
- Request-level field is exactly `requested_grid_level`; returned cells retain their actual `grid_level`, including mixed-level `minimal` covers.
- Native levels are Geohash `1..12`, MGRS numeric precision `0..5`, and ISEA4H resolution `0..15`; M2 exports and reuses `validate_requested_grid_level(grid_type, requested_grid_level)` with precisely those accepted inclusive ranges.
- MGRS `space_code` is canonical standard UTM/UPS MGRS; `topology_code` is `mgrs-topo-v1:<domain>:<level>:<canonical-space-code>` and is never presented as standard MGRS.
- MGRS boundary/topology geometry is the projected square clipped to the valid UTM/UPS domain; irregular cross-domain neighbor and child counts are valid.
- ISEA4H is ISEA, HEXAGON, PURE aperture 4, WGS84 authalic radius `6371.007180918475 km`, orientation `(11.25°, 58.28252559°, 0°)`, and public `space_code` is DGGRID v8.44 `SEQNUM` as an unpadded 1-based decimal string.
- ISEA4H runtime and tests must not import or invoke H3 or DGGRID. Fixed DGGRID vectors are mandatory; a live DGGRID re-check is optional after fixture generation.
- ISEA4H cell count at resolution `r` is exactly `10 * 4**r + 2`; exhaustive resolutions `0..6` contain `54,624` cells in total.
- Candidate count, output count, and elapsed-time limits return structured grid errors rather than partial output.
- Other packages consume `grid_core.sdk.CubeEncoderSDK`; no grid mathematics may be copied into `cube_split` or `cube_web`.
- Do not alter the M2 partition runner/result contract in M1. M2 adapts its result rows to `GridAddress.grid_level` and `GridAddress.topology_code` after this plan lands.
- Remove encoder S2, Tile Matrix, Plane Grid, and H3 implementation/dependencies only after all three replacement engines pass their narrow replacement gates.
- Every implementation slice uses an isolated worktree and creates exactly one reviewed temporary handoff commit. The integration branch applies the reviewed commit SHAs in the fixed order using `git cherry-pick --no-commit <SHA>`; it never uses `git merge --squash`, `git reset` to accumulate dirty changes, a push, a PR, or an assumed `cleanup` command.

### Public models owned by M1

`cube_encoder/grid_core/app/models/grid_address.py` owns:

```python
from pydantic import BaseModel


class GridAddress(BaseModel):
    grid_type: str
    grid_level: int
    space_code: str
    topology_code: str | None = None
```

`GridCell(GridAddress)` adds `center`, `bbox`, optional `geometry`, and `metadata`. `CompactGridCell(GridAddress)` adds `bbox`. The obsolete `level` and `cell_id` properties do not exist.

`cube_encoder/grid_core/app/models/request.py` exports the canonical range map and validator used by public request validation and by M2:

```python
LEVEL_RANGES: dict[GridType, tuple[int, int]] = {
    GridType.GEOHASH: (1, 12),
    GridType.MGRS: (0, 5),
    GridType.ISEA4H: (0, 15),
}


def validate_requested_grid_level(grid_type: GridType, requested_grid_level: int) -> int:
    minimum, maximum = LEVEL_RANGES[grid_type]
    if not minimum <= requested_grid_level <= maximum:
        raise ValueError(f"{grid_type.value} requested_grid_level must be in [{minimum}, {maximum}]")
    return requested_grid_level
```

All grid-bearing request models in `cube_encoder/grid_core/app/models/request.py` use `requested_grid_level` and call `validate_requested_grid_level`. M2 imports this function from that module rather than duplicating ranges. Topology and geometry operations consume an `address: GridAddress`, because an ISEA4H sequence number is meaningful only with its resolution and an MGRS topology result must retain both identities.

The SDK signatures are frozen as:

```python
def topology_code(domain: GridDomain, level: int, space_code: str) -> str:
    canonical = canonicalize_mgrs(space_code)
    value = f"mgrs-topo-v1:{domain.token}:{level}:{canonical}"
    if len(value.encode("ascii")) > 96:
        raise ValidationError("MGRS topology_code exceeds 96 ASCII bytes")
    return value
```

Canonicalization strips all whitespace, uppercases, parses the result with the standard library, and verifies that its digit suffix length is exactly `2 * grid_level`. Reject a canonical MGRS code when its precision differs from the explicit `grid_level`; for example, `31UDQ482511` is valid only at level `3` and its topology identity is exactly `mgrs-topo-v1:utm-31n:3:31UDQ482511`. Do not infer the topology domain only from the latitude-band letter; retain the point-selected or geometry-resolved UTM/UPS domain.

## Failure Blockers

- Any fixed-vector row differs, any Q2DI/SEQNUM reverse conversion fails, any resolution count differs from `10 * 4**r + 2`, or the `0..6` sum differs from `54,624`.
- ISEA runtime imports H3, invokes DGGRID, is not pure Python, or uses a non-DGGRID decimal `SEQNUM` as public `space_code`.
- MGRS returns an unclipped projected square outside its valid domain, accepts an MGRS numeric suffix whose length is not exactly `2 * grid_level`, emits a topology code that does not repeat the canonical `space_code`, treats point contact as an edge neighbor, assumes eight neighbors, or omits either standard/topology identity.
- `validate_requested_grid_level` is not exported from `grid_core.app.models.request`, accepts values outside Geohash `1..12`, MGRS `0..5`, and ISEA4H `0..15`, or M2 duplicates range logic instead of importing it.
- A `minimal` cover overwrites result `grid_level` with `requested_grid_level`.
- Legacy S2/Tile Matrix/Plane Grid/H3 encoder deletion occurs before all replacement gates pass.
- The configured real raster is absent or unreadable, the L4 harness is skipped, or the harness depends on M2 partition code.
- Any L1, L2, L3, or L4 command fails; any named review-role approval is missing; any high-severity review finding remains unresolved; or adversarial regression verification reproduces a correctness defect.
- Any temporary slice is integrated by `git merge --squash`, an accumulated dirty/reset workflow, or an order other than the reviewed fixed `git cherry-pick --no-commit` sequence; more than one M1 integration commit would be introduced; or any remote operation would occur without new user authorization.
