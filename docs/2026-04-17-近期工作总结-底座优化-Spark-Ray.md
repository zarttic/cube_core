# 近期工作汇报（简版）

更新时间：2026-04-17  
统计周期：2026-04-10 至 2026-04-16

## 一、阶段结论

本周已完成“底座优化 + Spark/Ray 并行化验证”的一轮闭环。  
当前结论是：小规模任务瓶颈主要在 Spark 固定开销，轻量并行（Local/Ray）可将剖分耗时稳定压到约 2 秒量级。

## 二、主要进展

1. 底座优化
- 完成 Geohash 内核向 S2 的切换（提交 `41d3d12`）。
- 三类引擎统一支持 `cover_compact`（仅返回 `space_code + bbox`），减少不必要几何构造。
- SDK/Service 补齐 `cover_compact`、`code_to_bbox` 能力，并增加缓存与输入校验。
- 补充对应回归测试与性能烟测项，确保优化后行为一致。

2. Spark 并行化探索
- 新增小数据优化策略（执行引擎切换、轻量参数、可跳过非关键统计）。
- 在 `data/landsat8`（8景）上，`level=7` 场景：
  - Spark 常规：`23.724s`
  - Spark 优化：`19.276s`
  - Local 轻量并行：`1.928s`

3. Ray 并行化探索
- 新增 Ray 版逻辑剖分作业，复用现有任务预处理链路，降低重复实现成本。
- 在同一数据集上：
  - Ray 首版：`2.873s`
  - Ray 自动参数：`2.003s`
  - 参数扫描最优：`1.941s`

## 三、问题与风险

1. 中大规模合成数据在当前机器上仍有 Spark 后期停滞现象（20景/80景场景）。  
2. 现有性能结论主要针对“小规模真实数据 + 本机环境”，集群侧还需补充压测验证。

## 四、下一步计划

1. 推进执行引擎自动路由（按任务规模在 `spark/local/ray` 间自动选择）。  
2. 对 `landsat8_synth_20/80` 做稳定性复测，定位 Spark 停滞根因。  
3. 统一 benchmark 指标与报告模板，纳入周期性回归。
