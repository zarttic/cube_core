# 剖分批次数据集合入库流程

## 业务流程

数据处理链路为：

`载入批次 -> 选择数据并剖分 -> 生成剖分批次数据集合 -> 质检 -> 数据入库`

一次剖分提交生成一个 `partition_run_id`，它代表一个剖分批次数据集合。集合可以合并同一
产品类型下多个来源载入批次，并包含多个数据集、景和数据单元。不同产品页面的待提交选择
彼此隔离，不会隐式进入同一个剖分批次。

## 手动入库

数据入库页面以剖分批次数据集合为主要操作对象。只有当前剖分版本存在、质检通过且尚未全部入库的集合才会出现在待入库队列中。

用户选择集合后，可以继续选择集合内的部分或全部波段数据单元进行入库。执行器仍按景合并读取同一份剖分产物以避免重复 I/O，但 `ingest_run_scenes.band_unit_ids` 固化本次选中的波段，完成、失败和重试只回写这些波段。

已完成入库的数据单元不会重复出现在待入库选择中；历史入库运行记录收纳在“入库记录”折叠区域中。

## 接口

- `GET /v1/ingest-runs/collections`：查询可操作的剖分批次数据集合及明细。
- `POST /v1/ingest-runs/collections/{partition_run_id}/ingest`：提交集合内选定的波段数据单元，请求体为 `{ "band_unit_ids": [...] }`。

质检完成不会自动创建入库任务，入库必须由数据入库页面手动提交。
