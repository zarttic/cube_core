# 待处理问题

## 部分数据重剖分会替换整个数据集当前输出

**状态**：待修复

**发现时间**：2026-07-18

**现象**

同一数据集包含多景、多波段数据时，只选择其中一景或部分波段重新剖分，新的输出版本会被设置为数据集的 `current_output_version`，旧输出版本则标记为 `superseded`。此时旧输出中仍存在的其他景/波段不能继续发布。

典型情况：

- 数据集同时包含 `LC08_L1TP_119030...` 和 `LC08_L1TP_120029...`；
- 重新剖分时只选择 `120029`；
- 当前输出版本只包含 `120029` 的波段；
- `119030-bqa` 仍在旧输出版本中，但旧版本已被标记为 `superseded`；
- 发布 `119030-bqa` 时返回 `publication target is not partitioned`。

**根因**

当前输出版本采用整体替换语义，重剖分任务不会自动合并未参与本次任务的既有景/波段结果。数据集级 `current_output_version` 指向最新一次任务的输出，而不是所有有效波段结果的合并快照。

**影响**

- 部分景或波段无法发布；
- 数据集概览可能出现当前输出与历史入库记录不一致；
- 用户需要重新选择整个数据集才能恢复完整发布范围。

**建议修复**

1. 将重剖分设计为增量合并：新任务只替换本次选择的景/波段，未选择的数据继续保留在新的有效输出快照中。
2. 发布目标校验应针对数据集当前有效输出集合，而不是单一任务输出。
3. 输出版本需要记录来源版本和被替换的数据单元，支持可追溯合并。
4. 在数据管理页面明确显示“本次重剖分范围”，避免用户误以为整个数据集都已重新生成。
5. 增加回归测试：多景数据只重剖分一景后，另一景已有的剖分、质检、入库和发布能力不能丢失。

在该问题修复前，重新剖分多景数据时应选择数据集内全部需要继续发布的景和波段。
