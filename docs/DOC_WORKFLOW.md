# Documentation Workflow

## Required updates per development cycle
1. Add one entry in `docs/DEVELOPMENT_LOG.md`.
2. If any bug/debugging happened, add one entry in `docs/BUG_LOG.md`.
3. Include exact verification commands and pass/fail result.
4. Keep entries chronological and immutable; append only.

## Naming rules
- Task IDs: `TASK-0001`, `TASK-0002`, ...
- Bug IDs: `BUG-0001`, `BUG-0002`, ...

## Quality bar
- Every entry must state:
  - what changed,
  - why it changed,
  - how it was validated.
