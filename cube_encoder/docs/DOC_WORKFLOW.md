# 文档维护流程

## 每个开发周期必须更新

1. 在 `docs/DEVELOPMENT_LOG.md` 追加一条开发记录。
2. 如果发生 bug 排查或修复，在 `docs/BUG_LOG.md` 追加一条记录。
3. 写明实际执行的验证命令和通过/失败结果。
4. 记录按时间顺序追加，历史记录不改写。

## 命名规则

- 任务 ID：`TASK-0001`、`TASK-0002` 等。
- Bug ID：`BUG-0001`、`BUG-0002` 等。

## 质量要求

每条记录必须说明：

- 改了什么。
- 为什么改。
- 如何验证。

新写文档默认使用中文。归档文档和历史流水账可以保留原始语言；若要重新发布为当前操作手册，
必须先整理为中文并更新入口索引。
