# 变更记录

本文记录 SDK/API 的重要变更。

## Unreleased

### M1 grid SDK breaking refactor (2026-07-14)

- **BREAKING**: production `grid_type` reduced to exactly `geohash | mgrs | isea4h`; removed `s2`, `tile_matrix`, `plane_grid` engines and the `h3`/`s2sphere` dependencies.
- **BREAKING**: request field renamed to `requested_grid_level`; returned cells carry their own `grid_level` (including mixed-level `minimal` covers).
- **BREAKING**: topology/geometry SDK and API operations now consume a `GridAddress` (`grid_type`, `grid_level`, `space_code`, optional `topology_code`) instead of a bare `level`/`code`.
- **BREAKING**: ST code prefixes are now `gh` (geohash), `mgrs` (mgrs), `i4h` (isea4h); removed `s2`/`tm`/`pg` prefixes.
- Added `GeohashEngine` (pure-Python base32, precision 1–12).
- Rewrote MGRS engine for UTM/UPS with a standard `space_code` plus a `topology_code` (`mgrs-topo-v1:<domain>:<level>:<space_code>`); geometry is the projected square clipped to the valid UTM/UPS domain.
- Replaced the H3-backed `isea4h` with a pure-Python implementation aligned to DGGRID v8.44 (ISEA projection, HEXAGON, PURE aperture 4, WGS84 authalic radius, orientation `(11.25, 58.28252559, 0.0)`); `space_code` is DGGRID `SEQNUM`. Runtime imports neither H3 nor DGGRID; fixed authoritative vectors are generated by DGGRID and committed as the test baseline.
- Fixed ISEA4H cover semantics: hierarchical positive-area intersection, complete-cell containment, and mixed-resolution `minimal` results preserve each cell's actual `grid_level`.

## [0.2.0] - 2026-03-11
- 新增可安装 Python SDK facade：`grid_core.sdk.CubeEncoderSDK`。
- 实现 `mgrs` 的 `cover_mode=minimal`。
- 为 `s2`、`mgrs`、`isea4h` 增加 minimal 合并降级行为。
- 增加日期变更线、分区边界和极区场景的引擎稳定性测试。
- 增加性能烟测 JSON 产物导出和 CI artifact 上传。
- 在 CI 中增加 SDK 包构建、wheel 安装和 sdist 安装烟测。

## [0.1.0] - 2026-03-11
- 首次增加可安装包元数据（`pyproject.toml`）。
- 核心能力：`s2`、`mgrs`、`isea4h` 格网、拓扑和 ST code API。
