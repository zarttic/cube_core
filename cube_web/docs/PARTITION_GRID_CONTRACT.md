# 剖分格网与方式契约

更新时间：2026-07-17

本文记录当前生产代码、API 和页面共同遵守的格网契约。历史实现仅在 Git 历史中保留，
不得作为新调用依据。

## 当前生产矩阵

| `grid_type` | 业务名称 | 剖分方式 | 原生层级 | Web 页面 |
| --- | --- | --- | --- | --- |
| `geohash` | 经纬度格网 | `logical` | 精度 `1..12` | 显示 |
| `mgrs` | 平面格网 | `logical` | 精度 `0..5` | 显示 |
| `isea4h` | 六边形格网 | `entity` | 分辨率 `0..15` | 显示 |

`s2`、`tile_matrix`、`plane_grid` 不属于当前生产契约，Web、strict API 和剖分作业不得
接受或重新暴露这些值。

## 方式分派

剖分方式由格网类型派生，不是独立业务选项：

```text
geohash -> logical
mgrs    -> logical
isea4h  -> entity
```

请求中的 `partition_method` 必须与派生值一致，否则后端返回校验错误。生产 Web 请求固定
使用 `cover_mode=intersect`、`time_granularity=day`、`max_cells_per_asset=0`；页面不显示
这三个高级字段。

`max_cells_per_asset=0` 表示不限制单数据单元的格网数量，不允许自动降低格网层级、静默
截断或改用其他覆盖方式。smoke、调试和故障复现可以显式使用小的正数。

## MGRS 语义

系统使用标准 UTM/UPS MGRS 编码，保留 zone、纬度带、100 km 方格和精度数字。它覆盖
全球有效坐标并支持跨 zone、反经线及 UTM/UPS 边界，但不伪装成 Geohash 式的全球连续
父子树。标准 MGRS `space_code` 是业务格网身份。

## 数据组织

载入批次用于记录一次载入动作，一个批次可以包含多个数据集，一个数据集可以包含多景，
景下保存资产和波段。后续剖分、质检、入库和发布以 `datasets` 中的数据集主记录为业务
入口；旧领域表暂时保留用于历史结果读取，不得为新载入创建第二条数据集主记录。

剖分结果提交成功后更新数据集当前输出版本，并记录剖分完成状态。质检通过后才能入库，
入库完成后才能发布。
