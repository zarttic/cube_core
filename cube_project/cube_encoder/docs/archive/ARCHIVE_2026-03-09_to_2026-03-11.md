# 项目阶段归档（2026-03-09 ～ 2026-03-11）

## 归档目的
本归档用于沉淀最近几天项目迭代成果，供后续研发、交接与评审使用。

## 阶段目标
- 形成可用的多引擎格网编码底座（Geohash / MGRS / ISEA4H-H3）。
- 打通 API 与 Python SDK 两种交付方式。
- 补齐拓扑元操作、最小覆盖策略、性能可观测与发布流程基础设施。
- 完成前端演示重构并接入真实后端能力。

---

## 一、核心能力建设

### 1) 多引擎格网能力完善
- Geohash：`locate / cover / topology` 能力持续完善，覆盖 `intersect/contain/minimal`。
- MGRS：补齐 `minimal` 覆盖模式，拓扑能力可用。
- ISEA4H：切换并稳定在 Uber H3 实现。

### 2) 最小覆盖（minimal）策略增强
- 对 `geohash/mgrs/isea4h` 增加跨层级 sibling 合并策略。
- 输出支持混合层级单元，降低结果规模。
- 补充 expanded-minimal 与 intersect 的一致性不变量测试。

### 3) 拓扑元操作能力增强
- 已支持 `neighbors / parent / children / geometry / geometries`。
- 前端元操作从“手输编码”切换为“选区驱动”：
  - 点选
  - bbox
  - polygon/rectangle 绘制
- 拓扑页新增独立格网类型选择。
- 取消页面初始化自动执行拓扑运算，改为用户触发后执行。

---

## 二、SDK 与交付能力

### 1) Python SDK 交付
- 新增 `CubeEncoderSDK` 统一入口（`grid_core/sdk/client.py`）。
- 覆盖格网、拓扑、时空编码核心方法。
- 形成可安装包基础（`pyproject.toml`）。

### 2) 版本与发布治理
- 建立 `CHANGELOG.md`。
- 建立 `docs/SDK_RELEASE.md`（SemVer 规则、发布检查项）。
- 包版本提升到 `0.2.0`。

### 3) CI 完善
- 性能烟测阈值集中配置：`.github/perf-thresholds.env`。
- `perf_smoke` 支持 JSON 结果导出并在 CI 上传产物。
- 增加 SDK 打包/安装烟测作业（wheel/sdist）。
