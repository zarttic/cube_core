# 无景化入库最小 DDL 与接口契约（V1）

更新时间：2026-04-21  
关联文档：`docs/2026-04-21-无景化遥感立方体入库方案.md`

---

## 1. 目标

给出可直接实现的最小落地规范：

- 3 张核心表 DDL
- 1 条 Ray 入库主流程接口契约
- 幂等与版本约束

说明：以下 SQL 以 PostgreSQL 语法为基线（控制面 + 索引面最小可跑）；后续迁移 Iceberg 时字段可保持一致。

## 2. DDL

## 2.1 Raw 资产表：`rs_raw_scene_asset`

```sql
CREATE TABLE IF NOT EXISTS rs_raw_scene_asset (
  id BIGSERIAL PRIMARY KEY,
  dataset TEXT NOT NULL,
  sensor TEXT NOT NULL,
  scene_id TEXT NOT NULL,
  band TEXT NOT NULL,
  acq_time TIMESTAMPTZ NOT NULL,
  raw_cog_uri TEXT NOT NULL,
  checksum TEXT,
  crs TEXT,
  resolution_x DOUBLE PRECISION,
  resolution_y DOUBLE PRECISION,
  bbox_min_lon DOUBLE PRECISION,
  bbox_min_lat DOUBLE PRECISION,
  bbox_max_lon DOUBLE PRECISION,
  bbox_max_lat DOUBLE PRECISION,
  cloud_cover DOUBLE PRECISION,
  quality_score DOUBLE PRECISION,
  version TEXT NOT NULL,
  run_id TEXT NOT NULL,
  ingest_time TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  metadata_json JSONB NOT NULL DEFAULT '{}'::jsonb,
  UNIQUE (scene_id, band, version)
);

CREATE INDEX IF NOT EXISTS idx_raw_asset_acq_time ON rs_raw_scene_asset (acq_time);
CREATE INDEX IF NOT EXISTS idx_raw_asset_run_id ON rs_raw_scene_asset (run_id);
CREATE INDEX IF NOT EXISTS idx_raw_asset_scene ON rs_raw_scene_asset (scene_id);
```

## 2.2 Cube 事实表：`rs_cube_cell_fact`

```sql
CREATE TABLE IF NOT EXISTS rs_cube_cell_fact (
  id BIGSERIAL PRIMARY KEY,
  grid_type TEXT NOT NULL,
  grid_level INT NOT NULL,
  space_code TEXT NOT NULL,
  time_bucket TEXT NOT NULL,
  band TEXT NOT NULL,
  st_code TEXT NOT NULL,
  cell_min_lon DOUBLE PRECISION NOT NULL,
  cell_min_lat DOUBLE PRECISION NOT NULL,
  cell_max_lon DOUBLE PRECISION NOT NULL,
  cell_max_lat DOUBLE PRECISION NOT NULL,
  value_ref_uri TEXT NOT NULL,
  source_scene_count INT NOT NULL,
  provenance_json JSONB NOT NULL,
  quality_rule TEXT NOT NULL,
  cube_version TEXT NOT NULL,
  run_id TEXT NOT NULL,
  ingest_time TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (grid_type, grid_level, space_code, time_bucket, band, cube_version)
);

CREATE INDEX IF NOT EXISTS idx_cube_fact_time_bucket ON rs_cube_cell_fact (time_bucket);
CREATE INDEX IF NOT EXISTS idx_cube_fact_band ON rs_cube_cell_fact (band);
CREATE INDEX IF NOT EXISTS idx_cube_fact_space_code ON rs_cube_cell_fact (space_code);
CREATE INDEX IF NOT EXISTS idx_cube_fact_run_id ON rs_cube_cell_fact (run_id);
```

## 2.3 任务控制表：`rs_ingest_job`

```sql
CREATE TABLE IF NOT EXISTS rs_ingest_job (
  job_id TEXT PRIMARY KEY,
  status TEXT NOT NULL,
  params_json JSONB NOT NULL,
  stats_json JSONB NOT NULL DEFAULT '{}'::jsonb,
  error_msg TEXT,
  retry_count INT NOT NULL DEFAULT 0,
  started_at TIMESTAMPTZ,
  finished_at TIMESTAMPTZ,
  output_snapshot TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_ingest_job_status ON rs_ingest_job (status);
CREATE INDEX IF NOT EXISTS idx_ingest_job_created_at ON rs_ingest_job (created_at);
```

## 3. 幂等写入 SQL 模板

## 3.1 Raw 资产 upsert

```sql
INSERT INTO rs_raw_scene_asset (
  dataset, sensor, scene_id, band, acq_time, raw_cog_uri,
  checksum, crs, resolution_x, resolution_y,
  bbox_min_lon, bbox_min_lat, bbox_max_lon, bbox_max_lat,
  cloud_cover, quality_score, version, run_id, metadata_json
) VALUES (
  :dataset, :sensor, :scene_id, :band, :acq_time, :raw_cog_uri,
  :checksum, :crs, :resolution_x, :resolution_y,
  :bbox_min_lon, :bbox_min_lat, :bbox_max_lon, :bbox_max_lat,
  :cloud_cover, :quality_score, :version, :run_id, :metadata_json
)
ON CONFLICT (scene_id, band, version)
DO UPDATE SET
  raw_cog_uri = EXCLUDED.raw_cog_uri,
  checksum = EXCLUDED.checksum,
  quality_score = EXCLUDED.quality_score,
  run_id = EXCLUDED.run_id,
  metadata_json = EXCLUDED.metadata_json,
  ingest_time = NOW();
```

## 3.2 Cube 事实 upsert

INSERT INTO rs_cube_cell_fact (

## 5. 融合规则契约（V1）

V1 固定：`best_quality_wins`

排序优先级：

1. `cloud_cover` 升序（越小越优）
2. `acq_time` 降序（越新越优）
3. `sensor_priority` 降序（由配置表提供）
4. `scene_id` 字典序（最终稳定 tie-break）

每条 cube 事实必须写入：

- `source_scene_count`
- `provenance_json`
  - `winner_scene_id`
  - `candidate_scene_ids`
  - `rule`
  - `scores`

---

## 6. 作业状态机

状态流转：

`pending -> running -> succeeded | failed`

失败重试：

- 同一 `job_id` 失败重跑时，`retry_count + 1`
- 保持 `cube_version` 不变，允许 `run_id` 变化
- 依赖 upsert 唯一键保证幂等

## 7. 验收 SQL（最小）

```sql
-- 1) 检查是否仍暴露景主键依赖（不应作为查询主键）
SELECT grid_type, grid_level, space_code, time_bucket, band, COUNT(*)
FROM rs_cube_cell_fact
GROUP BY 1,2,3,4,5
HAVING COUNT(*) > 1;

-- 2) 检查每条 cube 事实是否具备溯源
SELECT COUNT(*) AS missing_provenance
FROM rs_cube_cell_fact
WHERE provenance_json IS NULL OR provenance_json = '{}'::jsonb;

-- 3) 按 AOI 查询样例（需业务层把 AOI 映射为 space_code 集）
SELECT *
FROM rs_cube_cell_fact
WHERE time_bucket BETWEEN '2026-04-01' AND '2026-04-30'
  AND band = 'B04'
  AND space_code = ANY(:space_codes)
  AND cube_version = 'v1';
```
