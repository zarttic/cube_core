先确认你属于哪种使用方式

使用方式你需要启动什么适合谁
只看文档只启动本手册站业务人员、培训人员
本地页面开发Web API + Vue 前端；可以关闭认证前端开发、接口联调
真实生产运行Web API + 前端 + OpenGauss + Ray + MinIO + 认证实施和平台操作人员
!

本地页面能打开，不代表生产链路可运行。真正执行剖分必须检查 OpenGauss、Ray worker、MinIO 对象和认证。

前置检查

  1. 使用 Python 3.11：python3.11 --version。
  2. 确认仓库根目录下存在 cube_encoder/、cube_split/、cube_web/。
  3. 准备一个可以被 MinIO 读取的源对象。生产路径必须是 s3://cube/...，不要用某台机器的 /tmp 或本地绝对路径。
  4. 向管理员确认 OpenGauss、Ray、MinIO 和认证系统的运行时地址。地址属于部署配置，不要写到 manifest 或业务配置表。

创建 .cube_web.env

把下面模板复制到仓库根目录，再替换尖括号内容。文件已被 Git 忽略；不要执行 git add -f .cube_web.env，也不要把完整文件贴到工单或聊天记录。

CUBE_WEB_POSTGRES_DSN=postgresql://<user>:<password>@10.3.100.180:15400/postgres?client_encoding=UTF8
CUBE_WEB_RAY_ADDRESS=10.3.100.182:6379
CUBE_WEB_MINIO_ENDPOINT=10.3.100.179:9000
CUBE_WEB_MINIO_BUCKET=cube
CUBE_WEB_MINIO_ACCESS_KEY=<access-key>
CUBE_WEB_MINIO_SECRET_KEY=<secret-key>

读取顺序是：进程环境变量 → CUBE_WEB_ENV_FILE 指定文件 → 当前工作目录或仓库根目录的 .cube_web.env → 代码默认值。CUBE_WEB_POSTGRES_DSN 只是兼容变量名，实际目标是 OpenGauss。

启动后端

从仓库根目录启动，保持这个终端运行：

PYTHONPATH=cube_encoder:cube_split:cube_web \
python3.11 -m uvicorn cube_web.app:app \
  --host 0.0.0.0 --port 50039

另开终端检查健康接口：

curl -i http://127.0.0.1:50039/health

收到 HTTP 200 只说明 API 进程存活。若后续页面报数据库、Ray 或 MinIO 错误，继续检查对应依赖，不要把健康检查当成全链路验收。

启动前端

cd cube_web/frontend
npm install
npm run dev

前端默认运行在 http://localhost:50040，会把 /v1、/api 和 /health 代理到 50039。

认证模式

配置行为建议
未设置或 true前端按统一认证跳转；大多数 /v1/* 需要 Bearer Token生产必须使用
CUBE_WEB_AUTH_REQUIRED=0受控本地测试可以直接访问页面只用于本地测试，不要部署到生产
