现象可能原因正确处理

s3:// 对象排查

Ray worker 不能访问 driver 的本地临时路径。正确链路是 worker 读取稳定 s3 URI → 写入 worker 本地缓存 → 打开/转换 → 上传输出 → 用 s3:// 写 index。对象键包含中文时，先 URL decode，避免二次编码。

!

发生 ENOSPC 时，只清理当前 worker 的 /tmp/cube_split_source_cache 并重试一次。不能递归清理整个 /tmp，也不能删除其他任务目录。

质量错误的修复方向

错误类别责任方恢复动作
missing_crs / invalid_crs载入方修正影像元数据并重新交付
invalid_cog_uri / source_object_unreadable对象存储/载入方恢复对象、权限或 checksum 后重质检
output_count_mismatch / missing_tile_reference剖分运行原 partition run 的失败单元重试
carbon_coordinates / carbon_xco2_range碳数据提供方修复原始观测，不能在剖分层伪造值

如何收集证据

  1. 保存稳定业务 ID 和 UTC 时间。
  2. 保存错误码、rule_code、source_asset_id、band_code，不保存含凭据的原始异常。
  3. 保存输入对象 checksum 和目标 output_version。
  4. 保存重试前后的 attempt、质量序列和波段状态。
