为什么集合会出现在待入库

待入库接口只返回同时满足这些条件的剖分集合：

  • 存在 current output version；
  • 关联数据单元已经剖分完成；
  • 质量状态为 pass，或业务策略明确允许 warn；
  • 至少有一个波段尚未 completed；
  • 集合没有被取消或其他活动运行锁住。

接口：GET /v1/ingest-runs/collections。页面会把集合显示为“数据集数 · 景数 · 待入库波段数”。

页面操作步骤

1

刷新待入库集合

先刷新，不要使用上一轮打开页面时缓存的列表。列表中的 partition_run_id 是后续 API 路径参数。

2

选择集合

打开“选择数据入库”，选择一个 partition run。页面会加载该集合的 units。

3

选择 BandUnit

只选择质量 pass 或允许 warn 且 ingest_status 不是 completed 的单元。选择结果按 band_unit_id 固化。

4

提交入库

点击提交后返回 202，记录 ingest_run_id；随后在入库记录中观察波段级进度。

API 请求示例

POST /v1/ingest-runs/collections/RUN_20260719_EAST_001/ingest
Content-Type: application/json

{"band_unit_ids":["GF1-EAST-20200710-B01","GF1-EAST-20200710-B02"]}

入库状态怎么理解

状态含义可做什么
queued等待 worker等待，不要重复提交
running正在读取剖分产物并写入托管表等待或查看详情
completed该波段完成不能再次入库
partial_failure同一运行部分波段成功、部分失败只选择失败 band_unit 重试
failed所有相关波段失败或运行失败查看每个波段错误
cancelled未完成波段被取消按业务需要重新创建或使用允许的恢复路径

波段级重试

POST /v1/ingest-runs/{ingest_run_id}/retry
{"band_unit_ids":["GF1-EAST-20200710-B02"]}

重试接口要求明确传入失败 band_unit_ids。completed、queued、running 混入会返回 409；不存在或不属于该运行的波段返回 404。成功波段不应因为同一景的另一个波段失败而被回滚。

取消运行

POST /v1/ingest-runs/{ingest_run_id}/cancel
{"reason":"源对象维护，暂停本次入库"}

取消只会把未完成波段变为 cancelled，已 completed 的波段保持完成。对 completed、failed、cancelled 等终态运行再次取消会返回 409。

交接时看什么

在入库详情中保留 ingest_run_id、partition_run_id、每个 band_unit 的 status、error_message、attempt_count 和 retry_history。部分失败不是“整批失败”，不要直接删除历史运行。
