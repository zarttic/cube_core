领域对象关系

LoadBatch一次载入动作
Dataset长期管理主记录
Scene一次观测/场景
BandUnit可选择数据单元
Output某次剖分版本

一个 LoadBatch 可以包含多个 Dataset；一个 Dataset 可以在不同载入批次持续接收 Scene；一个 Scene 下有资产和 bands。剖分、质检、入库和发布以 Dataset 的当前输出为业务主线。

每个 ID 什么时候记录

ID产生时机后续用途
load_batch_id载入方交付导入幂等、批次查询、来源追踪
dataset_id导入登记数据集详情、质检、入库、发布
partition_run_id提交剖分剖分集合、质检集合、待入库集合
task_id提交异步任务queued/running/failed、取消、重试
output_version剖分输出质量和发布的版本身份
quality_run_id触发质检规则快照、结果、错误和 warn 审批
ingest_run_id手动入库波段级入库状态与 retry history
publication_id请求发布active/failed/withdrawn 版本历史

不要混淆三套状态

状态对象回答的问题错误例子
partition task剖分执行到哪一步任务 completed 不等于质量 pass
quality run当前输出是否满足质量门禁质量 pass 不等于已入库
ingest run选择的波段是否已写入托管数据入库 completed 不等于已发布
publication哪个版本对下游服务生效有 output 不等于 publication active

为什么 BandUnit 是关键

系统在业务上以 Scene 选择和物化资产，但质量、入库和重试需要精确到波段/变量。band_code 是稳定身份，band_name 只用于展示，band_type 区分 spectral、polarization、variable。前端不得根据 B01 猜传感器语义。

当前输出和历史输出

Dataset 的 current_output_version 指向默认服务对象。新的剖分成功后会产生新版本，历史版本保留用于审计；历史质量结果不能覆盖当前质量字段，历史输出也不能未经授权直接发布。
