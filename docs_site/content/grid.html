格网为什么会改变执行方式

logical 剖分以源影像窗口和逻辑格网索引为核心；entity 剖分需要生成实体瓦片几何并对观测/影像做实体覆盖。方式不是操作员可自由选择的参数，后端会校验格网与方式是否一致。

按需求选择

A

选 geohash

当下游以经纬度编码、区域查询和父子精度理解网格时使用。

B

选 mgrs

当下游明确要求标准 UTM/UPS MGRS。它不是 geohash 的另一种显示格式，不要假设存在连续全球父子树。

C

选 isea4h

当需要全球六边形邻接关系或实体瓦片时使用。它固定走 entity 剖分。

层级和资源

层级越细，覆盖单元数量、Ray 计算、MinIO 读写和索引规模通常越大。正式层级应结合源数据分辨率、AOI 面积、下游查询粒度和集群容量评估；不要为了让任务快一点静默降低层级。

!

smoke 任务可以使用 ISEA4H level 1、单景、ray_parallelism=2 和 max_cells_per_asset=50，但结果只能用于链路验证。

禁止值

生产 API、Web 和 split 不接受 s2、tile_matrix、plane_grid。遇到历史数据使用这些值时，先做迁移映射，不要重新暴露旧入口。
