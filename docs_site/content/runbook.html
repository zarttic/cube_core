B. 导入检查

  1. 记录 load_batch_id，确认它由载入方生成且可复用。
  2. 确认 datasets 非空；每个 Dataset 有 dataset_id、dataset_code、title、data_type。
  3. 确认每个 Scene 有稳定 scene_id、采集时间和空间信息。
  4. 确认每个资产有 checksum；非碳数据有 COG、bbox、crs；碳数据有 NetCDF/HDF5 source_uri。
  5. 确认 bands 非空且 band_code 稳定，不让前端猜波段语义。
  6. 导入后通过批次详情对账，不以 HTTP 202 单独证明数据完整。

C. 剖分检查

  1. 创建或确认 partition_run_id，并记录 source_batch_ids。
  2. 确认选中的 Scene 不重复，BandUnit 不重复。
  3. 确认本次运行的所有 Dataset 使用相同 grid_type 和 requested_grid_level。
  4. 确认格网方式：geohash/mgrs → logical；isea4h → entity。
  5. 确认 max_cells_per_asset=0 是正式任务；小正数只用于 smoke。
  6. 记录 task_id，轮询任务详情，不重复提交。

D. 质检和入库检查

  1. 只对 current output 发起质量运行，记录 quality_run_id。
  2. 查看 quality status 和 results_complete；error 或 incomplete 不能进入入库。
  3. 若为 warn，确认是否有批准人、理由和策略允许；不要把 warn 自动当 pass。
  4. 查看待入库集合，选择明确的 band_unit_ids。
  5. 记录 ingest_run_id；部分失败只重试失败波段。

绝对不要做的事

  • 不要把 DSN、Ray 地址、MinIO endpoint 或凭据写入 cube_web_configs。
  • 不要在生产启动时自动 seed 演示批次；只有演示环境显式设置加载开关。
  • 不要让 driver 生成只有本机可见的 /tmp COG 再交给 Ray worker。
  • 不要用 Path.exists() 判断 s3:// 输出；先下载到节点缓存再用 rasterio/netCDF 读取。
  • 不要为了重试创建新的平铺资产批次，优先使用原运行的正式 retry。
